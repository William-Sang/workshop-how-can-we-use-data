这是一个关于"数据"的趣味游戏，帮助我们快速相互了解。

  1. 每人领取3张便利贴。
  2. 写下3个关于你和数据的故事：2个真的, 1个假的。
  3. 轮流分享你的3个故事。
  4. 其他伙伴投票，猜出哪个是假的！

例如...

第二部分 (共识) (25分钟)

共识环节(3/4): 描绘理想未来

25:00

请大家思考...

想象一年后，当我们团队成功地"让数据用起来"时，公司里的**业务同事、领导、甚至客户**会如何评价我们？他们会说些什么？ 请用**一句简洁有力的话**，记录下那些最能体现我们团队**价值与成就**的评价。请从以下角度启发思考：

  • **业务增长**：数据如何直接驱动了收入、用户或市场份额的提升？
  • **效率提升**：数据如何简化了流程、节省了时间或降低了成本？
  • **决策优化**：数据如何让公司的决策更明智、更快速、更精准？
  • **团队影响力**：数据团队在公司中的地位和角色有何改变？

这些被想象出的"未来评价"，将是我们衡量成功、并最终向公司汇报价值的最有力证据！

第三部分 (50分钟)

聚焦真问题

50:00

方法学习

工具1: 用户旅程图 (User Journey Map)

这是一个"共情"工具，帮助我们站在用户（业务同事）的角度，完整地体验他们使用数据的全过程。

为什么用它？

  • 避免"我们觉得"的盲点，发现真实痛点。
  • 系统性地梳理所有用户触点，找到服务断点。
  • 为我们刚才提出的问题，提供用户视角的佐证。

如何操作

在白板上画一个表格，横轴是用户使用数据的阶段（如：产生疑问 -> 找数据 -> 分析数据 -> 形成结论 -> 应用决策），纵轴是：

  1. 用户行为 (Actions): 他做了什么？
  2. 想法感受 (Feelings): 他在想什么？感觉如何？
  3. 痛点 (Pain Points): 他遇到了什么困难？
  4. 机会 (Opportunities): 我们可以做什么？

方法学习

工具2: 5个为什么 (5 Whys)

这是一个挖掘问题根源的强大工具，防止我们只解决了表面症状。

核心思想

对一个问题，连续追问五次"为什么？"，层层递进，直到找到无法再问下去的根本原因。

如何操作

例：问题是"看板使用率低"

  1. 为什么？ → 因为数据不解决业务的实际问题。
  2. 为什么？ → 因为分析师不了解业务细节。
  3. 为什么？ → 因为分析师和业务缺少沟通机制。
  4. 为什么？ → ... (继续追问)

方法学习

工具4: "影响力-可行性" 矩阵

从众多想法中，快速识别出"必赢之战"的决策工具。

四象限分析

战略投入(高影响/低可行)
黄金方案(高影响/高可行)
避免(低影响/低可行)
价值有限(低影响/高可行)
← 可行性 →↑ 影响力 ↓

如何操作

  1. 在白板上画出四象限矩阵。
  2. 评估每个方案的"预期影响力"和"实施可行性"。
  3. 将方案便利贴放在矩阵的相应位置。
  4. 聚焦右上角的"黄金方案"！
